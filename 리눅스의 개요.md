#리눅스의 개요

## ⭐️ 하드웨어와 소프트웨어
```
하드웨어 : 컴퓨터에 붙어있는 실체가 있는 주변장치
ex) 컴퓨터 본체, CPU, 하드디스트, 마우스, 모니터 등

소프트웨어 : 쉽게말해 컴퓨터 프로그램
ex) 모니터 출력을 위한 프로그램, 마우스 움직임을 위한 프로그램
```

## ⭐️ 운영체제(OS)
```
- 컴퓨터 시스템의 메모리나 CPU같은 자원들을 효율적으로 관리
- 사용자가 컴퓨터를 편리하고 효율적으로 사용할 수 있도록 환경을 제공해주는 여러 프로그램의 모임
- 운영체제는 컴퓨터 사용자와 컴퓨터 하드웨어 간의 인터페이스로 동작하는 시스템 소프트웨어의 일종
- 다른 응용프로그램이 유용한 작업을 할 수 있도록 환경을 제공
```

## ⭐️ Linux의 역사
```
1969년 MIT, AT&T 벨 연구소의 켄 톰슨(Ken Tompson)이 초기 형태의 UNIX개발
후에, 필란드 헬싱키 대학의 리누스 토발즈(Lunus Torvalds)가 Minix의 커널 소스를 고쳐 GNU 시스템에 적합한 커널을 개발
```

## ⭐️ Linux와 다른 OS들의 차이점
```
Windows, Mac, Android, ios 등은 모두 일반인 즉, 개발자가 아닌 사람들을 대상으로 만든 운영체제이기 때문에 가능한 어려운 조작이 없음
요리로 비유하면 Windows나 Mac은 밀키트, 리눅스는 날재료
즉, 요리를 직접 함으로써 이것저것 자신만의 레시피대로 요리할 수 있음
```

## ⭐️ 오픈소스
```
리눅스는 오픈소스 기반
오픈소스(Open Source)란 누구나 무료로 가져다 쓸 수 있을 뿐만 아니라 그 코드도 마음대로 재조립을 할 수 있음
윈도우의 경우 정품 라이센스를 구입해야하는 유로 소프트웨어라는 점이 있어 일반적인 개발자로서는 뭔가를 개발해보기위한 진입장벽이 높음
반면 리눅스는 무료이기 때문에 얼마든지 서버에 설치하여 사용이 가능
필요에 따라 개조할 수 있기 때문에 이점이 많음
```

## ⭐️ Linux 장점
```
다중 사용자, 다중 작업이 가능한 시스템에 있어 서버 운영에 매우 적합
최고 권한자 ROOT사용자가 있고 이 사용자는 서버에 대한 모든 권한(서버를 변경, 조작, 파괴, 삭제 등) 보유
서버 규모가 점점 커지면 이 ROOT사용자는 직원들을 고용하여 서버를 같이 관리 
```

## ⭐️ Linux 구조
```
- 하드웨어 : 컴퓨터의 전자/기계장치의 몸체
- 커널 : 하드웨어와 소프트웨어의 연결 다리
- 쉘 : 커널과 사용자를 연결해주는 인터페이스(검은 배경에 흰 글씨로 이뤄짐)
- 응용프로그램 : 리눅스 내부의 응용프로그램
```
